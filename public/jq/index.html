jQuery

- 数多くのプラグインを提供するプラットフォーム
- JavaScript を補完する便利な機能を網羅
- jQuery での開発経験者が多い
- 多くのプラグインがメンテナンスされていない
- es6+ のスタイルに対応するためのドキュメントが少ない
- 必ずしも必要としない機能を含むため過剰なライブラリとなっている

CDN の利用

jQuery をホストしているCDNはいくつかありますが、レスポンスはcdn.jsdelivr.net が一番早いと思います。
 <script src="https://cdn.jsdelivr.net/npm/jquery@3.5.1/dist/jquery.min.js"></script>
code.jquery.com はあまり早くないので別のホストをおすすめします。

インストールと設定

# インストール
yarn add jquery

# jquery をimport する方法

# (1) jsファイルでimport する方法
import $ from "jquery";
window.$ = window.jQuery = $;
// slim 版のインポート
// import $ from "jquery/dist/jquery.slim.js";

# (2) webpack.config.js に記述する方法
# jsファイルでimport せずに$, jQuery を利用できる
  plugins: [
    new webpack.ProvidePlugin({
      $: "jquery",
      jQuery: "jquery"
      // slim 版を読み込む場合
      // $: "jquery/dist/jquery.slim",
      // jQuery: "jquery/dist/jquery.slim"
    })
  ],

jQuery のプラグインをインポートする場合、(1) の方法だとimport の順番を無視してjQuery 本体より先にインポートされ"jQuery is not defined" とエラーになることがあるので、jQueryの利用は(2) の方法がおすすめ。ただしwebpack から別のツールに移行するときにはちゃんと修正してください。
